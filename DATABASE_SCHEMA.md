# Job Finder App - Database Schema

## Collections & Documents

### 1. `users` Collection

**Purpose:** Store user profiles

**Document ID:** Firebase Auth UID

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| id | String | User ID (matches doc ID) |
| name | String | User's full name |
| phone | String | Phone number (formatted) |
| skills | String | Job seeker: skills; Employer: category |
| userType | String | "job_seeker" or "employer" |
| createdAt | Timestamp | Account creation time |

**Example:**
```json
{
  "id": "abc123def456",
  "name": "John Doe",
  "phone": "+1234567890",
  "skills": "Plumbing, Electrical, Carpentry",
  "userType": "job_seeker",
  "createdAt": "2026-04-01T10:30:00Z"
}
```

---

### 2. `jobs` Collection

**Purpose:** Store job listings

**Document ID:** Auto-generated by Firestore

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| id | String | Job ID |
| title | String | Job title |
| salary | String | Salary or rate (e.g., "$50/day") |
| phone | String | Employer's contact phone |
| category | String | "simple_jobs", "office_jobs", "online_jobs" |
| description | String | Job description (optional) |
| location | String | Job location (optional) |
| distance | Number | Distance in km (mock) |
| employerId | String | Reference to employer's user ID |
| createdAt | Timestamp | Job creation time |
| viewsCount | Number | Number of times viewed |
| contactsCount | Number | Number of applications |

**Example:**
```json
{
  "id": "job123abc",
  "title": "Plumber Needed for Kitchen Repair",
  "salary": "$50-100 per day",
  "phone": "+1234567890",
  "category": "simple_jobs",
  "description": "Need an experienced plumber for kitchen pipe repairs",
  "location": "Downtown, City",
  "distance": 2.5,
  "employerId": "abc123def456",
  "createdAt": "2026-04-01T09:15:00Z",
  "viewsCount": 45,
  "contactsCount": 8
}
```

**Indexes Needed:**
- Composite: `(category, createdAt DESC)`
- Composite: `(employerId, createdAt DESC)`

---

### 3. `applications` Collection

**Purpose:** Track job applications

**Document ID:** Auto-generated by Firestore

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| id | String | Application ID |
| userId | String | Reference to job seeker's user ID |
| jobId | String | Reference to job ID |
| status | String | "applied", "seen", "replied", "rejected" |
| createdAt | Timestamp | Application time |
| updatedAt | Timestamp | Last status update time |

**Example:**
```json
{
  "id": "app456xyz",
  "userId": "seeker789ghi",
  "jobId": "job123abc",
  "status": "replied",
  "createdAt": "2026-04-01T10:00:00Z",
  "updatedAt": "2026-04-01T14:30:00Z"
}
```

**Indexes Needed:**
- Composite: `(userId, createdAt DESC)`
- Composite: `(jobId, createdAt DESC)`

## Firestore Rules

```javascript
rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {
    // ===== JOBS =====
    match /jobs/{document=**} {
      // Public read
      allow read: if true;
      // Only authenticated users can create
      allow create: if request.auth != null;
      // Only owner can update/delete
      allow update, delete: if request.auth.uid == resource.data.employerId;
    }
    
    // ===== USERS =====
    match /users/{userId} {
      // Users can only read/write their own data
      allow read, write: if request.auth.uid == userId;
    }
    
    // ===== APPLICATIONS =====
    match /applications/{document=**} {
      // Users can only read their own applications
      allow read: if request.auth.uid == resource.data.userId;
      // Authenticated users can create applications
      allow create: if request.auth != null;
      // Only applicant can update their status
      allow update: if request.auth.uid == resource.data.userId;
    }
  }
}
```

## Query Patterns Used in App

```dart
// Get all jobs (limited to 50)
db.collection('jobs')
  .orderBy('createdAt', descending: true)
  .limit(50)
  .get()

// Get jobs by category
db.collection('jobs')
  .where('category', isEqualTo: 'simple_jobs')
  .orderBy('createdAt', descending: true)
  .limit(30)
  .get()

// Get employer's jobs
db.collection('jobs')
  .where('employerId', isEqualTo: userId)
  .orderBy('createdAt', descending: true)
  .get()

// Get user applications
db.collection('applications')
  .where('userId', isEqualTo: userId)
  .orderBy('createdAt', descending: true)
  .get()
```

## Performance Indexes

**Create these in Firestore for optimal performance:**

1. **Jobs by category & time**
   - Collection: `jobs`
   - Fields: `category` (Asc), `createdAt` (Desc)

2. **Jobs by employer & time**
   - Collection: `jobs`
   - Fields: `employerId` (Asc), `createdAt` (Desc)

3. **Applications by user & time**
   - Collection: `applications`
   - Fields: `userId` (Asc), `createdAt` (Desc)

4. **Applications by job & time**
   - Collection: `applications`
   - Fields: `jobId` (Asc), `createdAt` (Desc)

## Time-to-Live (TTL) Policies

**Inactive Jobs:** Delete after 90 days of no views/applications

**Rejected Applications:** Keep for 6 months then archive

**Implement via:**
- Scheduled Cloud Functions
- Or manual cleanup monthly
